Importance of deployment and production

In Week 4, I grasped the importance of development process of a website. Amongst three deployment process options, choosing to Work locally, Staging and then Production seemed to have more benefits over working on a direct live server or skipping the staging process. The staging process would generally help give feedbacks to the website before actual prodcution occurs.
In addition, I also understood the workflow involved in ensuring a smooth transition from development to deployment and production. It includes thorough testing, debugging and optimizing during staging process which reduces the errors before going for live production on the live server. Such kind of approach ensures a more stable and reliable website for users.

Deployment utilized for practical

To be frank, it was a difficult task to follow the above approach practically. On the other hand, I learnt to deploy my locally created WordPress website to a published web server or a live server. Using Local by Flywheel, generally made it easier to install WordPress in my local environment where I could develop my website. However, the locally developed website should be published to a live server for site production. There are various ways to deploy localhost site to a web server. I chose to use the All-in-One wordpress migration plugin offered by the WordPress itself where I exported my files for it to be sent. Then, CCP (Cloud Control Panel) cloudaccess site provided live server for dynamic sites allowing managing databases for the site. The exported files of my localhost site was then imported to the CCP cloudaccess. Hence, successfully deploying or producing my local site to a web server. However, some features are restricted to free users such as Github integration and database profiler.
After completing my practical, I wondered on how the staging process is actually working or seen during site development and production as I have not used Github version control. Since, this was an individual task, staging process must have passed unknowingly while working on my site.
