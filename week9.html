Learning Activities & Resources

In week 8, The learning activities generally involved understanding what Sass is and how it works. Sass stands for (Syntactically Awesome Style Sheets), it is a CSS preprocessor that allows for writing more maintainable and scalable CSS with features like variables, nesting, mixins and inheritance. By using Sass, developers can write more cleaner and more efficient CSS code.
After understanding what sass does, I have applied this knowledge to complete the practical 9. Typically, it invovled me to work on VS Code Editor and writing sass code that have the extension of .scss to demonstrate sass features.
Additionally, I also configured Gulp to automatically compile the process that also shows the user of the compilation. Running Gulp from terminal gives developers a visual changes and compiling of SCSS to CSS.
